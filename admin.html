Current Demo Storage

Local only
  • Products, categories, genres, tags, fixed page copy, and sort orders are saved to browser localStorage.
  • Thumbnail images are saved as browser data URLs for preview only.
  • CAD package files are not uploaded to ConoHa in this demo; the admin keeps the file name only.
  • The storefront reads the same localStorage key, so one browser can test “upload → display” immediately.

ConoHa Production Flow

Next build
  1. Admin login is handled by PHP sessions.
  2. CAD packages and thumbnails are uploaded to `public_html/dropworld.space/uploads/`.
  3. Product metadata is saved in ConoHa MySQL tables.
  4. The storefront fetches published products from a small PHP JSON API.
  5. Stripe handles checkout, payment status, and revenue reports.
